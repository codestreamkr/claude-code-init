# 20-report

이 단계에서는 현재 구조만 정리한다.
해결책이나 목표 구조를 먼저 쓰지 않는다.

## 반드시 확인할 것

- 외부 호출 시작 지점
- 벤더 클라이언트, 어댑터, 서비스 경계
- 리턴 URL, 웹훅, 배치, 재처리 진입점
- 결제 또는 연동 상태 저장 위치
- 외부 호출 전후 트랜잭션 경계
- 멱등 키, 중복 방지, 재시도 정책
- 로그, 추적 ID, 마스킹, 알림 구조
- 테스트 범위와 모킹 방식

## 리포트 출력 형식

1. 요청 분류
2. 현재 구조 요약
3. 벤더 종속 지점
4. 상태 저장과 전이 구조
5. 실패 경로와 운영 위험
6. 확인 필요

## 금지

- 목표 구조를 먼저 제안하지 않는다.
- 벤더 공식 문서로 확인되지 않은 필드나 상태를 단정하지 않는다.
- 구현 방안을 리포트 본문에 섞어 쓰지 않는다.
