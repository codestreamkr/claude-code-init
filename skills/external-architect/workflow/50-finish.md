# 50-finish

최종 단계에서는 결과를 하나의 문서로 닫는다.

## 통합 규칙

1. developer 산출물을 반영한다.
2. document 산출물을 반영한다.
3. 리포트, 제안, 실무 구체화를 문서 1개로 통합한다.
4. 공식 문서 미확정 항목을 `확인 필요`로 정리한다.
5. 문서 끝에 `## 이력관리`를 추가한다.
6. 최종 문서의 구조와 결론은 아키텍트가 확정한다.
7. developer 결과는 개발 관점의 프레임 정보로 반영한다.
8. document 결과는 문서 구조와 표현 정리 결과로 반영한다.
9. 두 agent의 산출물은 참고 입력이며, 최종 판단 책임은 아키텍트에게 있다.

## 역할별 요약

- 호페형: 체크리스트, 리포트, 제안, 위임 결과 검수, 최종 문서 통합
- developer: 개발 가이드 반영 내용 요약
- document: 문서 보완 반영 내용 요약

## 금지

- 위임 결과 없이 문서를 닫지 않는다.
- 최종 문서를 여러 초안으로 남기지 않는다.

## 최종 응답 형식

- 최종 응답에 `역할별 요약`을 포함한다.

1. 문서 위치
2. 반영 범위 요약
3. 확인 필요 또는 남은 위험
