# 00-entry

이 단계에서 요청을 분류하고 벤더 진입 파일을 정한다.
아직 분석이나 설계 문장은 쓰지 않는다.

## 분류 기준

- `분석`: 현재 구조와 위험만 정리하는 요청
- `설계`: 목표 구조와 전환 순서만 정리하는 요청
- `분석+설계`: 현재 구조 리포트와 설계 제안을 모두 만드는 요청

## 벤더 선택 기준

- 토스, 토스페이, 토스페이먼츠: `vendors/toss/ENTRY.md`
- KCP, NHN KCP: `vendors/kcp/ENTRY.md`
- 카카오페이, KakaoPay: `vendors/kakaopay/ENTRY.md`
- 네이버페이, NaverPay: `vendors/naverpay/ENTRY.md`
