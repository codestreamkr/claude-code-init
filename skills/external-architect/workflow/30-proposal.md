# 30-proposal

리포트가 끝난 뒤에만 이 단계를 실행한다.
이 단계에서는 목표 구조와 전환 기준만 정리한다.

## 제안에서 반드시 다룰 것

- 목표 구조와 책임 경계
- 벤더 원본 상태와 내부 도메인 상태의 분리 방식
- 승인, 취소, 조회, 웹훅/콜백 처리 원칙
- 멱등, 재처리, 중복 방지 기준
- 공존 기간 라우팅 기준
- 전환 순서와 롤백 기준
- 운영 체크포인트

## 제안 출력 형식

1. 목표 구조
2. 책임 분리
3. 상태 전이 원칙
4. 오류 처리와 재처리 원칙
5. 전환 순서
6. 확인 필요

## 금지

- 현재 구조 리포트 없이 이 단계를 시작하지 않는다.
- 현재 구조 설명을 다시 길게 반복하지 않는다.
- developer/document 위임 전에 최종 문서를 완료 처리하지 않는다.
- 정책 결정이 필요한 항목을 목표 구조, 구현 순서, 운영 체크포인트에 기본값처럼 쓰지 않는다.
