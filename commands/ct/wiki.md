# Confluence Wiki 문서 생성/변환

사용자 입력: $ARGUMENTS

## 작업 모드
1. 변환 모드(파일 경로 제공)
- 파일을 읽어 표준 템플릿 구조로 변환
- 원본 파일 덮어쓰기

2. 신규 생성 모드(텍스트 제공)
- 관련 코드 파일 탐색 및 읽기
- 표준 템플릿 채움
- `.0_my/wiki_[기능명].md` 저장

## 필수 준수 사항
1. 이모지 금지
2. 화살표 심볼 금지
3. 수평선(`---`) 금지
4. 코드 블록 규칙
- SQL/DDL: 코드 블록 유지
- Java/코드: 파일 참조(`ClassName.java:line`) 우선
- bash 명령: 필요 시만 코드 블록
5. 표/목록 중심 문서화

## 핵심 템플릿 섹션
1. 개요
2. Goals
3. 핵심 설계(결정/의존성)
4. 인덱스(해당 시)
5. API 명세
6. DTO 설계
7. 모니터링 및 측정
8. 프로퍼티 설정
9. 테스트
10. 부록(Alternatives Considered, 외부 연동)
